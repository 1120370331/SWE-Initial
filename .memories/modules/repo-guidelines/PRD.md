# repo-guidelines PRD

## 1. 背景与目标
- **业务背景**：仓库通过 AGENTS.md 指导协作代理执行日常任务，需有配套记忆文档以保持信息一致。
- **目标指标**：关键流程在记忆模块中有据可查，代理阅读后即可执行，无额外澄清需求。
- **成功判定**：新代理在首次任务中按照记忆文档即可完成作业，无漏读或误解反馈。

## 2. 用户画像与使用场景
- **核心用户**：驻场或远程的自动化/半自动化协作代理。
- **典型场景**：任务前查阅记忆模块了解流程；更新 AGENTS.md 后同步沉淀变更。
- **使用频次**：高

## 3. 范围与边界
- **In Scope**：
  - 仓库级别的操作约定与流程说明
  - 记忆文件目录与更新要求的维护
- **Out of Scope**：
  - 具体业务算法或代码实现细节
  - 数据管线的运行记录

## 4. 数据与集成
- **输入数据**：AGENTS.md 与仓库日常协作经验；文本格式，随变更即时更新，由维护者负责。
- **输出结果**：`.memories/modules/repo-guidelines/` 下的文档，供代理阅读。
- **系统依赖**：无外部系统依赖，仅依赖仓库本身。

## 5. 里程碑
| 阶段 | 时间 | 交付物 |
| --- | --- | --- |
| 规划 | 2025-11-01 | 明确仓库记忆模块结构 |
| 开发 | 2025-11-01 | 建立 repo-guidelines 模块并同步 AGENTS.md |
| 验证 | 2025-11-02 | 收集团队反馈并补充遗漏项 |

## 6. 风险与假设
- **关键假设**：协作代理会在任务前后遵照记忆流程；仓库更新节奏可在当天同步到记忆模块。
- **风险清单**：
  - 风险：AGENTS.md 快速迭代导致记忆不同步；缓解：设置更新流程提醒并在 PR 检查列表中勾选。
  - 风险：模块负责人缺席；缓解：指定备份维护者或团队轮值。
