# FUNCTION-MEMORIES-GOVERNANCE 说明

记忆文件治理流程，确保代理在任务前后按约定读取与更新 `.memories/`。

## 1. 定义
- **功能摘要**：指导代理如何使用与维护记忆文件夹，保持协作上下文同步。
- **入口位置**：`AGENTS.md::Memories 文件管理`
- **调用关系**：所有执行仓库任务的代理遵循此流程，并将更新结果写回 `.memories/modules/`。

## 2. 输入与输出
| 项目 | 描述 |
| --- | --- |
| 输入 | `.memories/modules/` 下对应模块的 `README.md`、`PRD.md`、`FUNCTION-*.md` |
| 输出 | 更新后的记忆文档，包含新的决策、假设与待办事项 |
| 副作用 | 触发 `modules/INDEX.md` 维护、创建新模块或补充 FUNCTION 文档 |

## 3. 实现要点
- 任务前必须阅读相关模块 `README.md`，按需补充 `PRD.md` 与 FUNCTION 文档。
- 任务结束后同步记录关键调整，必要时创建或扩展模块确保信息可检索。
- 新增模块需复制 `.memories/templates/module/` 模板并登记 INDEX。

## 4. 测试与验证
- **相关用例**：手动检查本次任务是否完成阅读与更新动作。
- **运行命令**：无自动化测试；在代码审查时检查 PR 描述是否引用记忆更新。
- **覆盖要点**：验证是否存在遗漏的决策记录与 INDEX 条目。

## 5. 数据与安全
- **敏感信息**：不记录个人敏感信息，若涉及请匿名化或改存外部系统。
- **容错策略**：若遗漏更新，下一次任务需补齐并在记忆中标注延迟原因。

- ❓ 是否需要自动提醒代理在工作流中确认记忆更新？
